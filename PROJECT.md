## 💡 Идеи
- statusline-режим (одна строка для tmux/zsh prompt)
- кэш сетевых лимитов с TTL, чтобы --watch не дёргал API каждый тик
- цветной календарь-хитмап (7×N) вместо одной полосы
- per-project разбивка для Claude (куда уходит больше)

## 📝 Заметки
- Данные читаются локально, токены вставлять не надо: Claude — Keychain `Claude Code-credentials`; Cursor — `cursorAuth/accessToken` в state.vscdb; Codex — роллауты.
- Стоимость всегда `~` (оценка): считаем и суб-агентов (sidechain) — это реальный расход против лимитов, поэтому чуть выше ccusage.
- Эндпоинты Cursor недокументированные — могут сломаться при обновлении Cursor, тогда панель честно покажет "n/a".
